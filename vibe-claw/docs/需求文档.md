# Vibe Claw 需求文档

> 本文档与 `docs/vibe-claw.md`、`docs/ACCEPTANCE.md` 对齐，描述当前 Vibe Claw 已拥有并需要持续保持的产品能力。更新时间：2026-04-29。

## 1. 产品定位

Vibe Claw 是独立的大模型与 Agent 能力平台，面向正式对外 SaaS/API 服务建设。它不是 Vibe IM 的子模块，也不是单纯聊天 UI，而是为 Vibe IM、第三方业务系统和自动化平台提供可复用、可审计、可治理的智能能力底座。

核心目标：

- 通过公开 `/v1` API 提供 Agent、模型、对话、协议运行、记忆、Run、Webhook、Token、用量和运维能力。
- 支持 DeepSeek、OpenAI-compatible 和 mock provider。
- 支持普通对话、SSE 流式对话、结构化协议运行和固定顺序多 Agent Run。
- 支持独立存储、审计、幂等、租户隔离、token scope、Webhook 签名和用量统计。
- 后台管理必须只通过公开 API 工作，不能形成绕过治理的第二套入口。

## 2. 当前已实现能力

### 2.1 服务与存储

- 独立 Fastify 服务，默认端口 `3100`。
- 独立后台 `/admin`。
- 独立 OpenAPI 契约 `/openapi.json`。
- 支持 `memory` 与 `postgres` 两种存储模式。
- 后台系统设置可配置存储模式，保存到 `.env.local`。
- 存储切换不做迁移：内存与 PostgreSQL 数据互不污染。
- 当前存储数据可通过后台危险操作重置。
- PostgreSQL 迁移覆盖 Provider、Agent、Conversation、Message、Memory、Protocol、Lease、Run、Queue、Token、Audit、Usage、Webhook、Idempotency 和 Conversation Lock。

### 2.2 模型 Provider

- 支持 `mock` provider。
- 支持 `openai-compatible` provider。
- DeepSeek 通过 OpenAI-compatible 配置接入。
- Provider 可创建、查询、修改。
- Provider 字段包含名称、类型、Base URL、默认模型、API Key Ref、重试次数、启用状态。
- API Key 只通过 `apiKeyRef`/环境变量/secret 引用，不在后台明文展示。
- 表格中敏感字段必须脱敏和单行省略显示。

### 2.3 Agent 管理

- 支持 Agent 创建、查询、修改、归档。
- Agent 可绑定 Provider 与默认模型。
- Agent 列表以表格展示，内容单行省略。
- 创建/修改使用按钮入口和弹窗。
- Agent 行操作可进入对应 Agent 的对话工作页与记忆工作页。

### 2.4 对话与会话

- `POST /v1/agents/:id/messages` 支持普通消息调用。
- `POST /v1/agents/:id/messages/stream` 支持 SSE 流式对话。
- 支持 conversation 创建、续聊、查询和消息列表。
- 同一 conversation 写入使用锁，避免并发消息顺序错乱。
- 支持 `compression` 参数：`none`、`recent_only`、`rolling_summary`、`hybrid`、`protocol_minimal`。
- 支持附件文本或附件元数据作为一次性上下文发送。
- 后台聊天 UI 为 Telegram 类聊天窗口：上方历史，下方附件、输入框、发送。
- 发送消息后本地先显示用户气泡和 Agent “正在思考”；请求失败在聊天内容中显示错误。
- 发送成功后清空输入框和附件；失败时恢复原输入文本。
- 聊天滚动遵循成熟聊天产品策略：用户在底部附近才自动跟随；打开会话和自己发送时定位到底部；不会因刷新状态导致回到顶部或弹跳。

### 2.5 记忆系统

- Agent 记忆支持写入、查询、状态更新。
- 记忆类型支持 `profile`、`semantic`、`episodic`、`working`。
- 记忆状态支持 `active`、`archived`、`rejected`。
- 记忆具备 tenant/project/agent 作用域。
- 普通对话可将相关 Agent 记忆注入上下文。
- 后台可从 Agent 行进入记忆工作页。

### 2.6 协议运行

- Agent 可注册协议：`POST /v1/agents/:id/protocols`。
- Agent 可查询协议：`GET /v1/agents/:id/protocols`。
- 协议运行：`POST /v1/agents/:id/protocol-runs`。
- 输入按 JSON Schema 校验。
- 模型输出按 JSON 解析并按输出 JSON Schema 校验。
- 校验失败返回明确 `valid:false`、`issues` 和稳定错误状态。

### 2.7 Run 与队列

- `POST /v1/runs` 创建异步 Run，返回 `202`。
- `GET /v1/runs` 查询列表。
- `GET /v1/runs/:id` 查询详情。
- `GET /v1/runs/:id/events` 查询事件。
- `POST /v1/runs/:id/cancel` 支持协作式取消。
- 支持单 Agent 与固定顺序多 Agent Run。
- PostgreSQL 队列支持 claim/lease、锁过期、重试、退避和 dead letter。
- 当前边界：尚未实现复杂并行 DAG、条件分支、自动 handoff 和人工确认节点。

### 2.8 Token、租约与权限

- Bearer token 鉴权。
- Token 支持 scope、tenantId、projectId。
- Token 支持创建、列表、撤销、轮换。
- Token 支持过期时间、IP allowlist、last used time、last used IP。
- 明文 token 只在创建和轮换时返回一次。
- 列表和后台不暴露 token hash 或明文。
- Agent lease 支持创建和查询，字段包括过期时间、调用次数、token 预算、允许协议。
- 关键写接口支持 `Idempotency-Key`，相同 key 不同 body 返回冲突。

### 2.9 Webhook、用量、计费和观测

- Run callback 支持 HMAC 签名。
- Webhook subscription 支持创建、查询、更新、事件匹配。
- Webhook delivery 支持投递日志、失败重试、dead letter 和手动 replay。
- 用量统计写入持久化 store，支持 `/v1/usage`。
- 计费摘要支持 `/v1/billing`，当前为最小账单/套餐闭环，未接入真实支付。
- 基础指标支持 `/v1/metrics`。
- Prometheus 指标支持 `/v1/metrics/prometheus`。
- 版本和兼容策略支持 `/v1/version`。
- 开发者文档索引支持 `/v1/developer-docs`。
- 审计事件支持 `/v1/audit-events`。

### 2.10 开发者接入

- OpenAPI JSON 可直接访问。
- 提供 `docs/developer-api.md`。
- 提供 API 版本策略 `docs/API_VERSIONING.md`。
- 提供变更日志 `docs/CHANGELOG.md`。
- 提供 Node SDK：`sdk/node/client.mjs`。
- 提供 Python SDK：`sdk/python/client.py`。
- 错误响应统一包含 `error`、`code`、`message`、`details`、`requestId`。

### 2.11 后台管理

后台页面包括：

- 概览。
- 模型配置。
- Agent 管理。
- Agent 对话工作页。
- Agent 记忆工作页。
- 调用记录。
- Token/租约。
- 系统设置。
- 开发者。
- 审计。
- 访问令牌弹窗。
- API 响应面板。

后台交互要求：

- 页面顶部不重复展示概览，只在概览页展示系统摘要。
- 所有表格内容单行显示，宽度不足省略。
- 所有创建/修改类操作统一使用按钮入口和弹窗。
- 表单保存前做字段校验，不符合时高亮具体字段。
- 重启服务和重置数据使用自定义确认弹窗。
- API 响应面板默认按用户上次状态隐藏或显示；复制按钮放在响应面板内部。

## 3. 当前明确边界

- 尚未实现复杂并行 DAG、多 Agent 条件分支、人工审批节点和自动 handoff。
- 尚未接入真实 Vault/KMS SDK；当前通过环境变量和 secret ref 约定实现密钥引用。
- 尚未接入真实支付、发票、扣费和客户自助开通。
- 计费/用量已有持久化聚合，但多实例强限额仍需要集中化限流或配额服务进一步增强。
- Provider 熔断、自动健康路由、Agent 版本灰度发布和质量评测门禁仍属于后续增强。

## 4. 非目标

- 不负责 Vibe IM 用户、好友、群聊、文件、消息投递和 IM 权限。
- 不直接读取或写入 Vibe IM 数据库。
- 不硬编码垂直行业业务流程。
- 不展示模型隐藏推理原文。
- 不允许 Agent 绕过平台 API 自行扩权或无限循环。
